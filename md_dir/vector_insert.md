vector_insert

# 문제이해
3가지 오버로딩 방식으로 insert를 진행한다.
2가지는 파리미터 수가 같다
- SFINAE로 처리 필요

크게 보면 방식은 2가지이다.
- 입력할 위치, 값을 "몇번" 넣을지, 넣을 값을 인자로 주는 경우
- 입력할 위치, inputIterator 시작 위치, inputIterator 끝 위치

# 문제재정의
동적 배열 특정 위치에 값을 집어넣는다.
집어넣는 방식은 크게 2가지이다. 
1. 입력위치, 입력크기, 입력값   
2. 입력위치, 입력할 반복자 시작위치, 입력할 반복자 끝 위치


# 문제해결방법
- SFINAE로 오버로딩이 바르게 되게 한다.
- 템플릿 오버로딩에 따라 처리될 수 있게 내부함수를 별도로 만든다.
- 방식이 크게 2가지로 나뉠 수 있다.
1. 입력할 크기가 capacity 내에서 입력 가능한지 아닌지
2. 입력 위치
    - 맨 앞
    - 중간 
    - 맨 뒤
- 입력위치에 따라 좌우되는 것은 맞는가?
capa를 미리 확보해준 이후에 진행한다면 어떻게 될것인가?
그렇게하면 복사하는 연산이 들어가야하는데
입력위치에 따라 복사해주는 부분이 달라질 수있을거 같다 생각했지만
다시 생각하니 그렇지 않을 것 같다.

capa를 미리 주고서 기존 배열 및 값을 넣어주어야하는 위치 등등을 for문 등을 이용해서 넣어준다.
한번에 처리 가능할 것으로 예상 
값을 넣어주는 부분은 uninitialized_copy()를 쓴다.
